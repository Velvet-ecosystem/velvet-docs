# Hosted Collaborator Boundary

Hosted assistants may help build Velvet. They are not Velvet.

They may:

- draft documentation
- review code and architecture
- scaffold tests and examples
- propose wording or structure
- help reason about implementation choices

They do not own:

- Velvet's identity
- continuity or lineage
- local memory
- Court authority
- capability tokens
- safety gates
- receipts
- physical presence
- hardware control

## Hosted Session Limits

A hosted model session may be stateless, externally operated, unavailable, or changed without the local installation's consent. Velvet therefore cannot treat hosted model continuity as system continuity.

## Safe Role

```text
hosted collaborator
  -> proposal or artifact
  -> local review and integration
  -> trusted repository or local Runtime
```

A hosted collaborator may write words or code. It does not run the sovereign system.

## Cloud-Optional Doctrine

Cloud services may improve drafting, research, or language range. Core identity, authority, safety, and execution must continue to function locally without them.

Cloud tools may help build the bridge. They are not the bridge's owner.
