# Velvet Docs

Canonical architecture, doctrine, contracts, deployment guidance, and ecosystem maps for Velvet AI.

Velvet is a local-first, offline-capable AI ecosystem designed to inhabit real bodies and surfaces without surrendering authority to hosted models or bypassing physical safety.

> Brain proposes. Court authorizes. Executors act. Receipts remember.

## Repository Purpose

This repository owns canonical ecosystem-level documentation.

It should contain:

- architecture shared across multiple repositories
- doctrine and authority boundaries
- end-to-end data and execution flows
- repository ownership maps
- deployment guidance spanning multiple components
- public diagrams and contributor orientation

It should not become a landfill for implementation notes. Repo-specific API details, tests, and module internals remain with their owning repositories.

## Core Laws

- API does not mean internet.
- Local observation does not equal authority.
- A receipt is evidence, not permission.
- Remote access may observe or request, but it never equals verified local physical presence.
- The language model may propose. It may not directly control shell, files, relays, CAN, actuators, or hardware.
- No valid receipt means no actuation.
- No verified physical presence means no deep privilege elevation.
- No trusted signature means no accepted update.
